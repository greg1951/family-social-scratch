BACKGROUND
- The founder features tab is a section within the application that allows founders to manage and access various features related to their account and business operations. This tab serves as a centralized hub for founders to easily navigate and utilize the tools and functionalities available to them.
- The founder features tab references four sets of forms: profile, settings, family and invites.
- The founder pages can be found in "src/app/(family)/family-founder-account" folder. 
- The founder tabs are implemented in "src/app/(family)/family-founder-account/page.tsx" and the components for the tabs are implemented in the "src/app/(family)/family-founder-account/founder-tabs.tsx" file.
- Two new tables have been implemented for feature configuration for a family.
  - The "featureRefererence" table contains a list of all of the "active" features that a family can configure.
  - The "familyFeatureConfig" table is a junction table that connects the family to the features that are active for that family. This table also contains a "status" column that indicates whether the feature is available for the family. 

REQUIREMENTS
- The purpose of this request is to implement a new tab in the founder's account page named "Features".
- The new "Features" tab should be added after the "Settings" tab and before the "Family" tab.
- The "Features" tab should display a list of features that are available to the family.
- If the family status is "trial" then by default all active features should be selected by the founder.
- The founder can toggle a feature off in this new tab which will set the status of that feature to "inactive" for that family in the "familyFeatureConfig" table. This means that the family will not have access to that feature.
- When the application main page is loaded, the family's selected features in the "familyFeatureConfig" table should be checked and only the features that are active for that family should be accessible in the application. 
  - For example, if the TV Shows feature is toggled off by the founder, then the TV Shows feature will not be visible on the main page; the main page header icons for TV Shows should be hidden as well.
