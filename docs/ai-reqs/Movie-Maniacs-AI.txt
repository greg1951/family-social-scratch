Movie Maniacs BACKGROUND
- The request will be to convert the existing Movie Maniacs home page to utilize the new movie schema that has been created and pushed to Neon via drizzle.
- The movie schema can be found in the "src/components/db/schema/family-social-schema-tables.ts" file. 
- The relevant tables have a "movie" prefix.
- The app router Movie Maniacs page and layout reside in "src/app/(features)/(movies)/movies" directory.
- The Movie Maniacs home page resides in the "src/features/movies/components/movie-home-page.tsx" file.

- The "movie-scroll-strip.tsx" file allows movie images and associated metadata to be displayed on the home page. The images for movies need to be stored and retrieved from AWS S3. 
  - Presently only member profile images are being retrieved from S3. These images reside in the "movies" folder in the bucket. 
  - The S3 bucket name is stored in the "familyS3Credentials" table in the "bucketName" column.
  - For testing purposes the "thosecrazyhughletts" bucket name on S3 will be used. 
  - The components to interface with S3 reside in the "src/api/s3-upload" folder.
  - The movie table itself has a column "movieImageUrl" which contains the full url of the image on S3. 
  - The recipe image should be stored in the "movies" folder in the bucket.
  - The memberId of the family member creating a recipe should be part of the recipe stored on S3, e.g. "memberId-63-movieTitle.jpg" where "movieTitle" is a compressed version of the value in the movie.movieTitle column.  
  - I have uploaded an image to the "thosecrazyhughletts/movies" folder on S3.
