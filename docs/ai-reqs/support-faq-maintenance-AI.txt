FAQ Maintenance - AI Requirements
Background:

This will begin the first phase of the Support implementatioin that will provide a Frequently Asked Questions (FAQ) page and form 
to enter new FAQs. This will be a simple implementation that will allow us to capture the most common questions and answers in one place,
to provide a way for users to see the FAQ.

Only a support admin can enter the FAQs. This permission is derived from the member table, isAdmin column.

- The initial FAQ maintenance home page begins with "src/app/(support)/(logged-in)/faq-maintenance/page.tsx".
  - There is a placeholder for this page which provides an example of how to enforce the isAdmin requirement to access the page.
- The generated FAQ maintenance page will be saved in "src/features/support/components/support-faq-update-home-page.tsx".
- The database tables reside in the "src/components/db/schema/support-faq-schema-tables.ts" file with a "support" prefix.
- The support tables have been pushed to the Neon database and are available for use.
- The generated database queries will be saved in "src/components/db/sql/queries-support.ts" file.

3. The FAQ form should utilize the TipTap rich text editor for the answer field to allow for formatting and styling of the FAQ answers
   - The FAQ answers are stored in the support_faq_qna table in the faqAnswerJson column.
4. Update the main-header DropdownMenu to add the Support/FAQ Maintenance link that is only visible to support admins.
4. The FAQs should be stored in the database and displayed on the FAQ maintenance page for editing and deletion.
5. Implement the necessary database tables and queries to support the FAQ maintenance functionality.
6. The page layout should conform to the same design system as the rest of the application, using the existing components and styles.
