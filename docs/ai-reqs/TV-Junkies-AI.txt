TV Junkies Background
- The request will be to convert the existing TV Junkies home page to utilize the new show schema that has been created and pushed to Neon via drizzle.
- The show schema can be found in the "src/components/db/schema/family-social-schema-tables.ts" file. 
- The relevant tables have a "show" prefix.
- The app router TV Junkies page and layout reside in "src/app/(features)/(tv)/tv" directory.
- The TV Junkies home page resides in the "src/features/tv/components/tv-home-page.tsx" file.

- The "tv-scroll-strip.tsx" file allows tv show images and associated metadata to be displayed on the home page. The images for recipes need to be stored and retrieved from AWS S3. 
  - Presently only member profile images are being retrieved from S3. These images reside in the "tv" folder in the bucket. 
  - The S3 bucket name is stored in the "familyS3Credentials" table in the "bucketName" column.
  - For testing purposes the "thosecrazyhughletts" bucket name on S3 will be used. 
  - The components to interface with S3 reside in the "src/api/s3-upload" folder.
  - The show table itself has a column "showImageUrl" which contains the full url of the image on S3. 
  - The recipe image should be stored in the "tv" folder in the bucket.
  - The memberId of the family member creating a recipe should be part of the recipe stored on S3, e.g. "memberId-63-showTitle.jpg" where "showTitle" is a compressed version of the value in the show.showTitle column.  
  - I have uploaded an image to the "thosecrazyhughletts/tv" folder on S3.
