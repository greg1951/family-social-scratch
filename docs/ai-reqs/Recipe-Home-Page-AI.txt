- The request will be to convert the existing Family Foodies home page to utilize the new recipe schema that has been created and pushed to Neon via drizzle.
- The schema can be found in the "src/components/db/schema/family-social-schema-tables.ts" file. 
- The relevant tables have a "recipe" prefix.
- The app router foodies page and layout reside in "src/app/(features)/(foodies)/foodies" directory.

- The "foodies-scroll-strip.tsx" file allows recipe images and associated metadata to be displayed on the home page. The images for recipes need to be stored and retrieved from AWS S3. 
  - Presently only member profile images are being retrieved from S3. These images reside in the "members" folder in the bucket. 
  - The S3 bucket name is stored in the "familyS3Credentials" table in the "bucketName" column.
  - For testing purposes the "thosecrazyhughletts" bucket name on S3 will be used. 
  - The components to interface with S3 reside in the "src/api/s3-upload" folder.
  - The recipe itself has a column "recipeImageUrl" which contains the full url of the image on S3. 
  - The recipe image should be stored in the "foodies" folder in the bucket.
  - The memberId of the family member creating a recipe should be part of the recipe stored on S3, e.g. "memberId-63-recipeTitle.jpg" where "recipeTitle" is a compressed version of the value in the recipe.recipeTitle column.  
  - I have uploaded eight images to the "thosecrazyhughletts/foodies" folder on S3.
