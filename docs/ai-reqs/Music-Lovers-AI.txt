Music Lovers Background
- The request will be to convert the existing Music Lovers home page to utilize the new music schema that has been created and pushed to Neon via drizzle.
- The music schema can be found in the "src/components/db/schema/family-social-schema-tables.ts" file. 
- The relevant tables have a "music" prefix.
- The app router Music Lovers page and layout reside in "src/app/(features)/(music)/music" directory.
- The Music Lovers home page resides in the "src/features/music/components/music-home-page.tsx" file.

- The "music-scroll-strip.tsx" file allows music music images and associated metadata to be displayed on the home page. The images for recipes need to be stored and retrieved from AWS S3. 
  - Presently only member profile images are being retrieved from S3. These images reside in the "music" folder in the bucket. 
  - The S3 bucket name is stored in the "familyS3Credentials" table in the "bucketName" column.
  - For testing purposes the "thosecrazyhughletts" bucket name on S3 will be used. 
  - The components to interface with S3 reside in the "src/api/s3-upload" folder.
  - The music table itself has a column "musicImageUrl" which contains the full url of the image on S3. 
  - The recipe image should be stored in the "music" folder in the bucket.
  - The memberId of the family member creating a recipe should be part of the recipe stored on S3, e.g. "memberId-63-musicTitle.jpg" where "musicTitle" is a compressed version of the value in the music.musicTitle column.  
  - I have uploaded an image to the "thosecrazyhughletts/music" folder on S3.
