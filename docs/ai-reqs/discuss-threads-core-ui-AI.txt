Background

The purpose of the discussion threads feature schema is to support discussion threads in most of the family social features.
TV, movies, music, books, poems, and even recipes are all things that people like to discuss with each other. 

Notes about the "discuss" tables
- The discussThread table will aggregate all of the discussion threads for all of the family social features.
  - The targetType column will store an type code that identifies the family social feature: "show", "movie", "music", "book", "poem", "recipe"
  - The targetId column will store the id of the specific item that the discussion thread is about. For example, if the targetType is "movie", then the targetId will store the id of the movie that the discussion thread is about.
  - The targetId column in a sense is a polymorphic foreign key implementation, when combined with targetType column.
- The "discuss" tables have been pushed to the database. 
- There is one record seeded in the discussThread table for testing purposes, for the TV show feature.
  - There are two records seeded in the discussPostReply table that reference the record in the discussThread table, for testing purposes.

Notes about the UI implementation plan
- For this initial UI implementation, we are focusing on identifying in the TV shows scroll strip any TV show that has a discussion thread associated with it.
- When a show has a discussThread associated with it, then render the lucide icon "message-square" in the top right corner of the image card for that show in the scroll strip.
- When the record is selected, provide a section in the show details below the Show Reactions section and above the Family Comments section.
- In the discussion section of the show details page, render the first name of the family member and the discussThreadTile and createdAt date.
  - Provide a "View Discussion" button in each discussion thread tile that will take the user to the discussion thread page for that show, where they can view all of the posts and replies in the thread, and add their own posts and replies to the thread.
  - For now, this button will not be functional, as the discussion thread page will be implemented in a later phase.
